# 开发阶段声明

**当前阶段：开发阶段** ← 可切换为"生产阶段"

## 开发阶段规则：
- **禁止**生成数据库迁移脚本，直接修改原始 schema/DDL 即可；
- **禁止**为 API 变更编写向后兼容代码，直接修改原接口定义；
- **禁止**保留旧字段/旧接口的兼容逻辑，假设所有调用方都会同步更新；
- 数据库结构变更时，直接输出完整的新 DDL/Entity 定义，无需考虑数据迁移；
- 默认假设：数据库可随时删除重建，无历史数据需要保留。

## 生产阶段（切换后启用）：
- 必须生成迁移脚本，保证数据不丢失；
- API 变更需向后兼容或版本控制。

## 核心约束规则
- 优先复用项目已有的核心服务、工具类、通用方法，禁止绕过现有模块从零编写重复逻辑；
- 操作数据库/缓存/第三方接口时，必须调用项目已有封装的 DAO/Repository/SDK 层，禁止直接写底层操作（如原生 SQL）除非用户明确要求，或者因为特殊原因必须直接操作；
- 不确定某功能是否已有实现时，先标注「待确认复用项：XXX」。

## 项目结构约定

- 项目根目录下必须有 `tmp` 文件夹
- 项目必须是 git 项目，若不是则先执行 `git init`
- `tmp` 目录必须被 `.gitignore` 忽略
- 所有总结性文档（修改总结、审查报告等）默认写入 `tmp` 目录

## 输出规范

1. 生成代码前，简要说明「核心复用项」；
2. 代码完成后，附「合规自查」：复用情况 / 命名目录约定 / 是否冗余；
3. 若有约束冲突，说明「约束适配调整」并给出最优解。
