# AI 开发交接文档目录

该目录用于存放按时间归档的开发交接文档，服务于以下场景：

- 在家开发到一半，切换到公司继续
- 切换电脑或开发环境后，希望新的 AI 会话快速接手
- 某次开发结束后，希望沉淀当前项目状态、关键决策、下一步动作与风险

## 文件结构

- **最新入口文件**：项目根目录下的 `AI_HANDOFF.md`
- **历史归档文件**：`docs/ai-handoff/YYYY-MM-DD-HHmm-主题.md`

## 命名规则

历史归档文件统一使用以下命名格式：

```text
YYYY-MM-DD-HHmm-主题.md
```

例如：

- `2026-04-01-0028-用户认证模块.md`
- `2026-04-01-2315-支付状态机整理.md`
- `2026-04-02-0910-错误码与异常处理.md`

## 使用建议

1. 每次准备中断开发、切换开发地点、切换设备或切换 AI 会话前，生成一份新的交接文档
2. `AI_HANDOFF.md` 只保留最新交接入口和摘要
3. 详细历史保存在本目录下，便于回溯
4. 交接文档应重点包含：
   - 当前目标
   - 已完成内容
   - 当前做到哪里
   - 未完成事项
   - 下一步最小可执行动作
   - 风险与阻塞点
   - 优先阅读文件

## 目标

通过标准化交接文档，避免在不同地点、不同设备、不同 AI 会话之间反复重新理解整个项目，提高开发连续性与 AI 协作效率。
