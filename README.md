# AI 提示词规则仓库

一套用于规范 AI 编程助手行为的提示词规则集合。

## 目录结构

```
├── CLAUDE/
│   ├── CLAUDE.md                           三阶段工作流、八荣八耻
│   ├── agents/                             智能体定义
│   │   ├── api-docs-extractor.md           API文档提取器（子代理版）
│   │   └── api-docs-extractor-simplified-cn.md  API文档提取器（通用版）
│   └── profiles/                           用户能力档案
├── 代码规则/
│   ├── LLM优先编码原则.md
│   ├── 代码审查规则.md
│   └── 代码编写规则.md
├── 分支规则/
│   └── debug-log.md                        Debug 日志规范
├── 文档规则/
│   ├── AI编程文档规范.md
│   └── 代码对齐文档规则.md
└── tools/
    ├── api-crawl-prompt.md                 API 文档爬取任务模板（Ocean Engine专用）
    └── api-crawl-prompt-general.md         API 文档爬取任务模板（通用版）
```

## 核心内容

### AI助手核心规则
- **三阶段工作流**：分析问题 → 制定方案 → 执行方案
- **八荣八耻**：AI编程行为准则

### 代码审查规则
- **审查分级**：L1快速扫描 / L2重点审查 / L3全面审查
- **重复代码**：3次以上才提取
- **反模式识别**：上帝类、长函数、深嵌套等

### 代码编写规则
- **LLM优先编码原则**：与 AI 协同开发的最佳实践
- **开发阶段约束**：开发阶段 vs 生产阶段
- **项目结构约定**：tmp目录、Git初始化

### 分支规则
- **debug-log.md**：Debug 日志规范

### API文档提取工具
- **api-docs-extractor-simplified-cn.md**：通用API文档提取器规范
  - 支持有/无分类两种模式
  - 三层验证机制确保准确性
  - 适用于任何API文档提取场景
- **api-crawl-prompt-general.md**：通用API文档爬取任务模板
  - 可直接复制使用的提示词模板
  - 支持灵活的目录结构和命名规则
  - 包含多种使用场景示例

### AI编程文档规范（7大原则）
1. 信息密度优先
2. 模块化原则（<500行/文档）
3. 单一职责原则
4. 精准相关原则
5. 可视化优先原则
6. 完全自洽原则
7. 交叉引用原则

## 使用方式

- **维护规则**：直接编辑对应目录下的文件
- **AI使用**：将对应文件直接添加到系统提示词
