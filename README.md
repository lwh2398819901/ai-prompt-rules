# AI 提示词规则仓库

一套用于规范 AI 编程助手行为的提示词规则集合。

## 目录结构

```
├── human-readable/            人类友好版（源文件，用于维护）
│   ├── CLAUDE/
│   │   ├── CLAUDE.md          三阶段工作流、八荣八耻
│   │   ├── agents/            智能体定义
│   │   └── profiles/          用户能力档案
│   ├── 代码规则/
│   │   ├── 代码审查规则.md
│   │   └── 代码编写规则.md
│   └── 文档规则/
│       ├── AI编程文档规范.md
│       └── 代码对齐文档规则.md
│
└── ai-readable/               AI友好版（用于系统提示词）
    ├── CLAUDE/
    ├── 代码规则/
    └── 文档规则/
```

## 双版本设计

| 版本 | 目录 | 用途 | 特点 |
|------|------|------|------|
| 人类友好版 | `human-readable/` | 维护和阅读 | 简洁、中文、易理解 |
| AI友好版 | `ai-readable/` | 系统提示词 | 结构化、强调词、条件逻辑 |

**工作流**：人类编辑 `human-readable/` → 转换生成 `ai-readable/`

## 核心内容

### AI助手核心规则
- **三阶段工作流**：分析问题 → 制定方案 → 执行方案
- **八荣八耻**：AI编程行为准则

### 代码审查规则
- **审查分级**：L1快速扫描 / L2重点审查 / L3全面审查
- **重复代码**：3次以上才提取
- **反模式识别**：上帝类、长函数、深嵌套等

### 代码编写规则
- **开发阶段约束**：开发阶段 vs 生产阶段
- **项目结构约定**：tmp目录、Git初始化

### AI编程文档规范（7大原则）
1. 信息密度优先
2. 模块化原则（<500行/文档）
3. 单一职责原则
4. 精准相关原则
5. 可视化优先原则
6. 完全自洽原则
7. 交叉引用原则

## 使用方式

- **维护规则**：编辑 `human-readable/` 目录下的文件
- **AI使用**：将 `ai-readable/` 目录下的文件添加到系统提示词
