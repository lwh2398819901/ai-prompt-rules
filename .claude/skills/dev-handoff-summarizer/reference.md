# 开发交接技能参考规范

## 目标

本技能用于把当前项目状态沉淀为跨地点、跨设备、跨 AI 会话可续接的上下文文档。

## 强制原则

- **真实性优先**：只能基于真实代码、文档、差异、搜索结果和用户说明总结
- **可执行优先**：要让下一位 AI 能继续干活，而不是只看懂
- **搜索优先于总结**：默认先搜索，除非用户明确要求跳过
- **最小必要阅读**：必须列出优先阅读文件，不要让下一位 AI 重新扫全项目
- **风险显式化**：风险、阻塞、依赖和待确认问题必须单独列出
- **双文件输出**：同时输出 `AI_HANDOFF.md` 与历史归档文档

## 历史归档路径

```text
docs/ai-handoff/YYYY-MM-DD-HHmm-主题.md
```

## 最新入口路径

```text
AI_HANDOFF.md
```

## 搜索要求

当用户未明确禁止搜索时，至少完成以下工作：

1. 搜索与当前主题相关的代码和文档
2. 识别核心入口文件与关键调用链
3. 找出当前任务主要变更点
4. 识别未完成事项、TODO、潜在风险和上下文缺口
5. 判断是否存在重复设计、过时结构、命名不一致、过度复杂调用

## 禁止事项

- 禁止编造完成情况
- 禁止假设用户已确认某项技术决策
- 禁止只写空泛总结，不写下一步动作
- 禁止让下一位 AI 自行重读整个项目
- 禁止省略风险、阻塞和待确认问题
- 禁止把不确定内容写成确定事实
